# Тестове завдання — Junior Fullstack Developer (Fintech)

Привіт! Це невелике практичне завдання для позиції **Junior Fullstack Developer** у
fintech-команді. Перед тобою спрощена система внутрішніх P2P-переказів між
рахунками на **Next.js + PostgreSQL + Prisma**.

Код зараз "у проді": на демо він начебто працює, але від користувачів уже були
скарги, а в базі з'явилися дивні баланси. Твоє завдання — розібратися чому.

> Це завдання свідомо містить помилки. Ми хочемо побачити не «ідеальний код», а
> як ти читаєш чужий код, знаходиш проблеми й акуратно їх виправляєш.

---

## Стек

- **Next.js** (App Router), React, TypeScript
- **PostgreSQL** + **Prisma ORM**
- Бізнес-логіка — у Server Actions

## Ключові файли

- `src/app/actions/transfer.ts` — функція переказу (головне, що треба дивитися)
- `src/lib/auth.ts` — імітація сесії (зараз "залогінений" Alice / user-1)
- `src/lib/prisma.ts` — Prisma-клієнт
- `prisma/schema.prisma` — схема БД
- `prisma/seed.ts` — тестові дані
- `scripts/repro.ts` — заготовка скрипта для відтворення бага

---

## Як запустити

Потрібні **Node.js 18+** і **Docker**.

```bash
# 1. Залежності
npm install

# 2. База даних
cp .env.example .env
docker compose up -d

# 3. Схема + тестові дані
npm run db:reset      # prisma db push + seed

# 4. Застосунок
npm run dev           # http://localhost:3000
```

Тестові рахунки:

| Рахунок     | Власник | Баланс   | Користувач |
|-------------|---------|----------|------------|
| `acc-alice` | Alice   | 1000 USD | user-1 (ти)|
| `acc-bob`   | Bob     |  500 USD | user-2     |
| `acc-carol` | Carol   |    0 EUR | user-3     |

Скрипт відтворення бага:

```bash
npm run repro
```

## Завдання

### Частина 1. Аналіз (~15 хв)

Уважно прочитай `src/app/actions/transfer.ts`. Знайди помилки й **опиши кожну** у
файлі `ANSWERS.md`:

- що не так (1–2 речення);
- наскільки це критично для фінансової системи (Critical / High / Medium / Low);
- що може статися з грошима користувачів через цю помилку.

Впорядкуй знайдене за пріоритетом і поясни, чому саме так.

### Частина 2. Виправлення (~30 хв)

Виправ функцію `transferMoney`, щоб переказ був коректним. Зверни увагу на:

- **Валідацію суми** — сума має бути додатною та коректним числом;
- **Достатність коштів** — не можна переказати більше, ніж є на рахунку;
- **Переказ самому собі** — з рахунку на той самий рахунок робити не можна;
- **Валюту** — не змішуй різні валюти (USD → EUR без конвертації);
- **Авторизацію** — переказ може ініціювати лише власник рахунку-відправника
  (поточного користувача бери з `auth()`);
- **Обробку помилок** — якщо переказ не вдався, функція не повинна повідомляти
  про успіх.

**Обов'язково:** доповни `scripts/repro.ts`, щоб він показував проблему **до**
виправлення й підтверджував, що **після** фіксу вона зникла.

### Частина 3. Короткі питання (~15 хв, письмово)

Дай відповідь **на 2 з 3** питань у `ANSWERS.md`:

1. Чому баланс краще зберігати не у форматі `Float`? Як би ти зберігав гроші?
2. Два перекази з одного рахунку відбуваються "одночасно". Що може піти не так і
   як це можна не допустити? (достатньо ідеї, без реалізації)
3. Як би ти покрив функцію переказу тестами? Які 3–4 випадки перевірив би
   в першу чергу?
